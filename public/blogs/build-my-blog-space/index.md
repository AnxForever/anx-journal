# 从模板到自己的博客

这篇文章是这个站点真正意义上的起点。

前面花了不少时间做结构清理：删掉样板文章、重做首页和归档、把站点从“开源项目演示”改成“自己的长期空间”。现在终于可以从第一篇正式文章开始，把这个地方慢慢写成自己的样子。

## 为什么还要自己做博客

原因很简单：

- 我想保留一个不被平台节奏裹挟的输出空间
- 我希望技术、设计、产品判断能放在同一个语境里
- 我希望写作能和项目、收藏、笔记一起沉淀，而不是分散在很多地方

博客对我来说不是“发长文的地方”，而是一个持续整理自己的方式。

## 这次重做想解决什么

这个站最开始只是一个可用的模板，但模板只能解决“搭起来”，不能解决“像不像我”。

所以这一轮调整，核心不是换个配色或者删几张图，而是把几个更本质的东西重新定下来：

1. 首页先表达“我是谁，我在关心什么”
2. 文章页更像阅读器，而不是单纯的 Markdown 容器
3. 归档页默认服务阅读，管理能力退到次级入口
4. 项目、分享、关于这些页面，要和博客属于同一个世界，而不是独立功能页

这也是我想继续坚持写博客的原因：网站本身，就是表达的一部分。

## 之后会写些什么

目前先给自己留三条主线：

### 1. 建站与重构记录

包括这次博客改造、信息架构、界面取舍、内容组织方式，以及后续一些更细节的实现记录。

### 2. 技术笔记

不追求面面俱到，更偏向那些真正值得记下来、以后还会回看的问题和解决方式。

### 3. 随笔和观察

有些内容不一定适合写成教程，但值得留下来，比如产品判断、设计感受、互联网观察，或者做项目时的一些阶段性想法。

## 接下来准备补的内容

这篇发出去之后，下一批内容大概会是：

- 博客这次改造里做对和做错的地方
- 我希望长期维护怎样的分类体系
- 为什么我更喜欢“作者站”而不是“文章列表页”
- 一些真正能代表我关注方向的技术文章

如果你之后翻到这里的时候，发现这个站已经和现在不一样了，那应该说明它确实在往前长。

这比一开始就“做完”，更重要。
